# Demo Script

## Goal

Give judges one clean watch-through they can score immediately.

Core line to repeat:

`Initia Raid Club is the clearest demo of Initia-native gaming UX: identity, bridge, auto-signing, and fast onchain action in one loop.`

## Format

- Total length: `90-120 seconds`
- Recording style: one smooth pass, minimal mouse movement, no dead time
- Order: `identity -> bridge -> ticket -> raid -> settlement -> leaderboard`

## Shot List

### 0:00 - 0:10

**Screen**
- Landing screen
- Logo and hero visible
- Cursor already near the main CTA

**Voiceover / subtitles**

`Initia Raid Club is the clearest demo of Initia-native gaming UX: identity, bridge, auto-signing, and fast onchain action in one loop.`

`This is a short-session boss run built on a dedicated Initia MiniEVM rollup.`

### 0:10 - 0:25

**Screen**
- Connect wallet
- Claim handle
- Show the claimed identity in the UI

**Voiceover / subtitles**

`The loop starts with identity.`

`The player claims a handle once, and that identity carries into the profile, combat log, and leaderboard.`

### 0:25 - 0:40

**Screen**
- Open bridge flow
- Bridge in INIT
- Mint a raid ticket
- Enable auto-signing if that control is visible in the flow

**Voiceover / subtitles**

`Next comes onboarding.`

`The player bridges in funds, enables session UX, and mints a ticket without leaving the product.`

### 0:40 - 0:50

**Screen**
- Enter the featured boss lane
- Let the boss screen settle for a second so the UI is readable

**Voiceover / subtitles**

`Now the player enters the featured encounter: Nightglass Warden.`

`The fight is tuned for a three-beat demo clear: guard, attack, and finish.`

### 0:50 - 1:10

**Screen**
- Execute first action: guard
- Execute second action: attack
- Execute third action: special
- Keep the auto-signing status area visible during the actions

**Voiceover / subtitles**

`This is the core demo moment.`

`Auto-signing lets the player chain actions quickly, so the onchain flow feels like gameplay instead of wallet management.`

### 1:10 - 1:25

**Screen**
- Show settlement screen
- Highlight coins, XP, and loot

**Voiceover / subtitles**

`The run settles immediately into visible rewards: coins, XP, and loot.`

`The result is not abstract. It feeds progression right away.`

### 1:25 - 1:40

**Screen**
- Return to base
- Open activity or leaderboard
- Show the updated result in profile or ranking

**Voiceover / subtitles**

`That outcome flows back into the player profile, the combat history, and the leaderboard.`

`The whole product is built around one repeatable consumer loop.`

### 1:40 - 1:55

**Screen**
- Pull back to a stable overview: hero, raid screen, or home base
- End on a strong branded frame

**Voiceover / subtitles**

`Initia Raid Club is the clearest demo of Initia-native gaming UX: identity, bridge, auto-signing, and fast onchain action in one loop.`

## Recording Notes

- Keep the video to one path only. Do not open extra tabs or explain architecture on-screen.
- If a transaction takes time, leave the status visible instead of talking over it.
- Do not show setup friction, terminal windows, or unfinished admin steps.
- If the live rollup path is not stable enough for recording, use the cleanest product path available and keep the narration focused on the user loop.

## Optional Closing Line

Use this only if you still have a few seconds left:

`This project shows why Initia fits consumer apps: identity, onboarding, and repeated onchain actions all stay inside one product flow.`

## Backup Talking Points

- Combat actions map to real `MsgCall` transactions in onchain mode.
- The repo includes both the frontend and the MiniEVM contract.
- The demo is intentionally narrow so judges can verify product value in one watch.
